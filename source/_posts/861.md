> 上次杜老师讲了"培训归来 3 步收纳"的方法，不少小伙伴留言问：alias 到底怎么起名字，才不会跟系统命令打架？今天杜老师就把命名约定这件事单拎出来讲一讲。

<!-- more -->

## 一、为什么命名约定重要

杜老师一开始是这种写法：

```
alias k='kubectl'
alias kg='kubectl get'
alias kgp='kubectl get pod'
```

用了不到一周就发现：

* 敲 `k` 经常跟某些命令冲突
* 自己都记不清哪个 alias 对应哪条长命令
* 别人用电脑时一脸茫然

后来杜老师定了一套约定，再没出现过这些问题。

## 二、杜老师的 3 条命名约定

### 约定 1：所有 alias 加统一前缀

杜老师用的是 `x` 前缀：

```
alias xk='kubectl'
alias xkg='kubectl get'
alias xkgp='kubectl get pod'
```

好处有两个：

* 不会跟系统命令冲突
* shell 按 `x` 找，能一键列出自己的所有 alias

您可以选自己的前缀，但**一定保持一致**。

### 约定 2：短前缀按"频次"分层

杜老师的分层是这样的：

* 每天用：`xk` `xd` `xg`（2 个字母）
* 每周用：`xkg` `xdc`（3 个字母）
* 偶尔用：保留长命令，不加 alias

这样按 `x<1字符>` 就能列出高频命令。

### 约定 3：动作在前，对象在后

杜老师的命名顺序是 **"工具 + 子命令"**：

```
xkgp  ← kubectl get pod
xkga  ← kubectl get pod -A
xdc   ← docker compose
xdcl  ← docker compose logs -f
```

不是 `xkpg`，也不是 `kgp`。

按这种顺序，敲到第二个字母基本能猜出完整 alias。

## 三、杜老师自己的 `.zshrc.local` 节选

```
# === kubectl ===
alias xk='kubectl'
alias xkg='kubectl get'
alias xkgp='kubectl get pod'
alias xkga='kubectl get pod -A'

# === docker ===
alias xd='docker'
alias xdc='docker compose'
alias xdcl='docker compose logs -f'

# === 自学 ===
alias xv='vim'
alias xll='ls -lah'
alias xg='git'
alias xgs='git status'
```

注意：高频命令才加 alias，一个月用不到 3 次的保持长命令。

## 四、约定一旦定下来，别轻易改

杜老师自己踩过这个坑：

* 前缀从 `x` 改成 `m`，结果老的 alias 全部失效
* 命令顺序从"工具+子命令"改成"子命令+工具"，每次都敲错

约定一旦定下来，至少用半年再考虑调整。
