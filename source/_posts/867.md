> 杜老师参加过几次技术培训，最早一次是全抄命令，抄完基本记不住。后来改成"只记关键字"，反而效果好得多。今天杜老师就把这个方法分享给大家。

<!-- more -->

## 一、为什么不全抄

杜老师一开始的笔记是这样的：

```
# CentOS 7 链路聚合
nmcli connection add type team con-name team0 ifname team0 autoconnect yes config '{"runner": {"name": "activebackup"}}'
nmcli connection add type team-slave con-name team0-1 ifname eth1 master team0
nmcli connection modify team0 ipv4.method manual ipv4.addresses 192.168.1.250 connection.autoconnect yes
nmcli connection up team0
...
```

看起来很完整，但杜老师复盘下来发现：

* 抄完之后基本没看过
* 真要用的时候不知道在哪一页
* 命令细节记不清，关键字反而不记得

## 二、关键字笔记长什么样

杜老师现在的笔记是这样的：

```
# CentOS 7 链路聚合

team0 — 创建虚拟网卡，绑定成员用
team-slave — 向虚拟网卡添加成员网卡
ipv4.method manual — 手动配置地址
teamdctl team0 state — 查看聚合状态

注意：activebackup 模式两边不要有空格
```

每一条只记**这条命令是干什么的**，外加一句触发场景。

## 三、只记关键字的 3 个好处

### 好处 1：笔记变短，能跟得上

杜老师自己测试过：

* 全抄命令：跟不上讲师节奏
* 只记关键字：基本能跟上，偶尔还能追问

注意：跟不上的部分可以录音，回头补笔记。

### 好处 2：回看时一眼能找到

关键字笔记的特点是：

* 每条不超过 2 行
* 关键词明显，扫一眼就能定位

杜老师现在回看培训笔记，基本 1 分钟就能找到想要的。

### 好处 3：强迫自己真正听懂

只记关键字有一个隐性要求：

* 必须真正听懂这条命令是干什么的
* 不懂就没法写出一句话总结

这个"强迫自己理解"的过程，比记下完整命令更有价值。

## 四、杜老师自己的笔记模板

```
# [主题]

[命令关键字1] — [一句话功能]
[命令关键字2] — [一句话功能]
[命令关键字3] — [一句话功能]

注意：[关键坑点]
```

培训结束后，杜老师会再做两件事：

* 把关键字笔记转成 markdown，放到 `~/notes/training/` 目录
* 在 `.zshrc.local` 里给高频命令加 alias
