> 杜老师经常出差，酒店 Wi-Fi 不稳这事基本每次都遇到。这次，杜老师把自己用过且还在用的 3 个 fallback 工具分享给大家。

<!-- more -->

## 一、为什么需要 fallback

杜老师自己碰到过几次：

* 酒店 Wi-Fi 看着满格，但 SSH 连不上
* 视频会议开到一半断流
* 关键文件同步不上去

总不能直接认栽，备一套 fallback 工具很必要。

## 二、3 个 fallback 工具

### 工具 1：手机热点

杜老师用的最多。手机开热点，让电脑直接连。

* 优点：基本能在所有酒店场景下立刻切换
* 缺点：流量消耗大，长时间会议不适合

杜老师建议：

* 关键演示、首次连入酒店网络之前，先备好热点
* 必要时把手机热点当"主用"，酒店 Wi-Fi 当"辅助"

### 工具 2：随身 Wi-Fi 设备

杜老师有台 4G 随身 Wi-Fi：

* 优点：流量便宜，可以同时给多个设备用
* 缺点：得提前充电，出门前要确认是否有电

适合酒店会议这种"明确需要稳定 1-2 小时"的场景。

### 工具 3：手机 USB 共享网络

iPhone 和 Mac 之间可以做到：

* 插线即用
* 不需要密码
* 比热点耗电少

杜老师偶尔调试网络时会用上。

## 三、杜老师自己的组合

```
酒店 Wi-Fi 正常 → 用 Wi-Fi
酒店 Wi-Fi 不稳 → 切手机热点
关键演示 → 提前用 USB 共享测试
多人协作 → 随身 Wi-Fi
```

这套组合，杜老师用了一年多，目前没掉过链子。
