> 杜老师经常出差，每次高铁上的几个小时是难得的整块时间。但车上电源、网络不稳定这事几乎每次都会遇到，今天杜老师就把自己用了一年多的 3 个 fallback 分享给大家。

<!-- more -->

## 一、为什么需要 fallback

杜老师自己碰到过几次挺尴尬的情况：

* 笔记本电池剩 30%，前座乘客又把插座拔了
* 手机想连车上 Wi-Fi，但网页根本打不开
* 演示用的 PDF 还没同步到本地，4G 信号又掉了

总不能直接认栽，备一套 fallback 工具很必要。

## 二、3 个 fallback 工具

### 工具 1：充电宝 + 双口充电线

杜老师用的是 20000 mAh 的充电宝，配合一根 USB-C 转 USB-C + Lightning 的二合一线：

* 优点：可以同时给笔记本和手机供电
* 缺点：体积略大，建议放在双肩包外层

杜老师建议：

* 出门前一定充满，出差当天早上再补一次
* 高铁上发现插座被占，第一时间切到充电宝

### 工具 2：手机热点 + USB 共享网络

iPhone 和 Mac 之间可以做到：

* 插线即用，不消耗手机电量
* 比开热点稳定，不会突然断流
* 适合需要稳定网络的演示场景

杜老师偶尔需要在车上回一封带附件的邮件，就直接 USB 共享。

### 工具 3：本地资料预先同步

这一条严格说不算工具，但杜老师觉得是性价比最高的一招：

* 出门前 30 分钟，把当天可能用到的资料同步到本地
* 演示文件、PDF、代码片段，全部放进一个 `出差-2026-XX-XX/` 目录
* 信号再差，本地永远能打开

## 三、杜老师自己的组合

```
笔记本没电 → 用充电宝补
需要发邮件 → USB 共享网络
演示当天 → 前一晚就把资料同步到本地
```

这套组合，杜老师用了一年多，目前没掉过链子。
